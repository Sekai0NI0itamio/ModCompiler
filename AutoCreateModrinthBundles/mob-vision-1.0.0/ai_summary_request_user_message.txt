# Mob Vision

Mob Vision lets you set how far each hostile mob can see and chase players. You edit a simple config file to change detection ranges, or stop certain mobs from targeting players at all. The changes take effect immediately without restarting the game.

## Description

Mob Vision lets you configure the detection range for each hostile mob type. Adjust the range in the config file `mobvision.txt` or set a range to 0 to stop a mob from targeting players. The mod updates every second so changes apply while you play.

### How It Works (for the user)
- Hostile mobs within the configured range will chase and attack the nearest player.
- You can adjust each mob’s sight range in the config file `mobvision.txt`.
- Setting a mob’s range to 0 stops it from targeting players.
- The mod checks for changes once per second, so updates happen while you play.
- It works on all loaded hostile mobs such as zombies, skeletons, spiders, and creepers.
- No new items, blocks, or commands are added.

### Features
- Configurable detection ranges per mob type.
- Ability to disable targeting for specific mobs by setting range to 0.
- Automatic reload of the config file when it is edited during gameplay.
- Supports vanilla hostile mobs only; no extra content is added.
- Simple text‑based configuration that can be edited with any editor.

        String mobId = getMobId(mob);
        double configuredRange = config.getRangeForMob(mobId);
        IAttributeInstance followRange = mob.getEntityAttribute(SharedMonsterAttributes.FOLLOW_RANGE);
        if (followRange != null && Math.abs(followRange.getBaseValue() - configuredRange) > 0.001D) {

                continue;
            }

            bestDistanceSq = distanceSq;
