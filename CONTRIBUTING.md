# Contributing to ModCompiler

Thanks for your interest in contributing.

## 1) File issues first
Open an issue first for bugs or feature requests so we can align on scope.

## 2) Fork and branch workflow
- Fork the repository.
- Create a focused branch from `main` (for example: `fix/issue-123` or `feat/new-feature`).
- Open a pull request with a clear summary of what changed and why.

## 3) Code style and tests
Follow the existing style in this repository and run checks before opening a PR:

```bash
./gradlew check
```

If the target area uses other repository tests, run those as well.

## 4) Commit messages
Use clear, imperative commit messages (for example: `Fix null check in version resolver`).

## 5) License of contributions
By submitting a pull request, you agree to license your contribution under this project's MIT License.

## 6) Communication
Use GitHub Discussions or issues for questions and proposals, and tag @Sekai0NI0itamio when needed.
