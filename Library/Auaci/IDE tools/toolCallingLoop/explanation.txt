# Tool Calling Loop - Detailed Explanation

## Overview
The Tool Calling Loop is a framework that handles the execution of tools, collects results, and feeds them back to the AI model. It manages iterative tool execution for agent tasks and builds feedback context for model consumption.

## Purpose
To provide a structured way to execute multiple tools in sequence, track their results, and present that information to the AI model for continued reasoning and decision-making.

## Configuration
This is a JavaScript class that is instantiated with options:
- `maxIterations` (optional, number): Maximum number of tool call iterations (default: 10)

## Core Components

### Classes
1. **ToolResult**: Represents the result of a tool execution
   - Properties: toolName, input, output, error, timestamp, success
   
2. **ToolCall**: Represents a tool call request from the model
   - Properties: id, toolName, input, timestamp
   
3. **ToolCallingLoop**: Manages iterative tool execution
   - Properties: options, toolCalls, toolResults, conversationHistory, maxIterations, currentIteration

## Logic Flow

### ToolCallingLoop Constructor
- Initializes with provided options
- Sets up empty arrays for tracking calls and results
- Sets maxIterations (default 10)
- Initializes currentIteration to 0

### executeToolCall Method
Executes a single tool call:
1. Validates that an executor function is provided
2. Calls the executor with toolName and input
3. On success: Creates ToolResult with output and success=true
4. On error: Creates ToolResult with error and success=false
5. Records the result in toolResults and toolCalls arrays
6. Returns the ToolResult

### processToolCallRound Method
Executes multiple tool calls in sequence:
1. Iterates through provided toolCalls array
2. For each toolCall, calls executeToolCall
3. Collects all results in an array
4. Increments currentIteration
5. Returns array of ToolResult objects

### buildFeedbackContext Method
Builds context for model feedback:
1. Returns object with:
   - totalCalls: Number of tool calls made
   - totalResults: Number of tool results received
   - successCount: Count of successful tool executions
   - errorCount: Count of failed tool executions
   - currentIteration: Current iteration number
   - maxIterations: Maximum allowed iterations
   - toolResults: Array of all ToolResult objects
   - toolCalls: Array of all ToolCall objects

### formatToolResults Method
Creates readable format for including in next prompt:
1. Iterates through toolResults
2. For each result:
   - Adds `<tool_result tool="${toolName}">` tag
   - If success: Adds output (stringified if object)
   - If error: Adds error message
   - Optionally adds error recovery suggestion for common errors
   - Closes with `</tool_result>` tag
3. Returns formatted string

### shouldSuggestErrorRecovery Method
Determines if error recovery should be suggested:
1. Checks if error message contains common recoverable patterns:
   - 'missing required', 'required property'
   - 'validation failed', 'invalid'
   - 'not found', 'file not found', 'path not found'
   - 'no such file', 'permission denied'
2. Returns true if any pattern matches

### shouldContinue Method
Determines if tool execution should continue:
1. Returns true if currentIteration < maxIterations

### reset Method
Clears state for new conversation:
1. Empties toolCalls and toolResults arrays
2. Clears conversationHistory
3. Resets currentIteration to 0

### getExecutionSummary Method
Returns summary text of tool execution:
1. Builds feedback context
2. Returns formatted string with success/failure counts

## Integration with IDE
- Used by the AI agent to execute tools in sequence
- Results are fed back to the model for continued reasoning
- Error recovery suggestions help the model self-correct
- Iteration limits prevent infinite loops
- Execution summary provides metrics for debugging

## Best Practices
1. **Set Appropriate Max Iterations**: Based on task complexity (default 10 is usually sufficient)
2. **Handle Errors Gracefully**: Use error recovery suggestions when provided
3. **Reset Between Conversations**: Call reset() for each new task/conversation
4. **Monitor Iterations**: Use getExecutionSummary() to track progress
5. **Leverage Feedback Context**: Provide buildFeedbackContext() to the model for informed decisions

## Technical Implementation
The tool calling loop:
1. Decouples tool execution from tool definition
2. Provides standardized result tracking
3. Enables iterative agent behavior
4. Facilitates model-tool-model feedback loops
5. Supports error recovery and self-correction

## Usage Example
```javascript
const { ToolCallingLoop } = require('./toolCallingLoop');

// Create loop instance
const loop = new ToolCallingLoop({ maxIterations: 5 });

// Define tool executor function
async function executor(toolName, input) {
  // Call actual tool implementation
  return await actualToolExecutor(toolName, input);
}

// Process tool calls from model
const toolCalls = [
  new ToolCall('1', 'view', { path: '/src/main.js' }),
  new ToolCall('2', 'str_replace', { 
    path: '/src/main.js', 
    oldStr: 'oldCode', 
    newStr: 'newCode' 
  })
];

// Execute round
const results = await loop.processToolCallRound(toolCalls, executor);

// Build feedback for model
const feedback = loop.buildFeedbackContext();
const formattedResults = loop.formatToolResults();

// Check if should continue
if (loop.shouldContinue()) {
  // Continue with next iteration
}

// Get summary
const summary = loop.getExecutionSummary();
```

## Error Handling
- Individual tool execution errors are caught and wrapped in ToolResult
- Error recovery suggestions are automatically generated for common errors
- The loop continues unless maxIterations is reached
- Executor failures don't break the loop - they're recorded as failed results

## Benefits
1. **Structured Execution**: Organized way to run multiple tools
2. **Result Tracking**: Complete history of all tool calls and results
3. **Model Feedback**: Structured format for feeding results back to AI
4. **Error Recovery**: Automatic suggestions for common mistakes
5. **Iteration Control**: Prevents runaway tool execution
6. **Conversation Isolation**: Reset capability for separate tasks
