# Edit Lines Tool - Detailed Explanation

## Overview
The Edit Lines tool provides precise line-based editing capabilities, allowing users to edit, insert, or delete specific lines by line number. It offers fine-grained control over file modifications using 1-indexed line numbers.

## Purpose
To make targeted changes to files by specifying exact line numbers, enabling precise insertions, replacements, and deletions without relying on content matching.

## Configuration
- **Tool Name**: `edit_lines`
- **Version**: 1.0.0
- **Execution Mode**: optional
- **Parameters** (all required unless noted):
  - `path` (string): Path to the file
  - `start_line` (number, required): 1-indexed starting line number
  - `end_line` (number, optional): 1-indexed ending line number (inclusive). Default: same as start_line
  - `new_content` (string, optional): New content to insert/replace. Empty or omit for delete action.
  - `action` (string, optional): Action to perform - enum: ["replace", "insert_before", "insert_after", "delete"]
    - Default: "replace"

## Logic Flow
1. Validate that the file exists and is readable
2. Convert line numbers to 0-indexed for internal processing
3. Validate line numbers are within file bounds (1 to total_lines)
4. If end_line not provided, set end_line = start_line
5. Validate start_line <= end_line
6. Based on action parameter:
   - `replace`: Replace lines from start_line to end_line (inclusive) with new_content
   - `delete`: Remove lines from start_line to end_line (inclusive)
   - `insert_before`: Insert new_content before start_line
   - `insert_after`: Insert new_content after end_line
7. If new_content is empty/omitted and action not specified, default to delete
8. Write the modified file back to disk

## Key Features
- **Precise Line Control**: Exact line number specification
- **Flexible Actions**: Replace, insert before/after, delete
- **Range Operations**: Support for single lines or line ranges
- **Default Behavior**: Sensible defaults for missing parameters
- **Boundary Handling**: Proper handling of file start/end

## Error Handling
- **Retryable**: No (fail-fast on errors)
- **Common Errors**:
  - out of range: Line numbers exceed file length → Use `view` to see file length
  - file not found: File does not exist → Use `find_files` to locate file

## Usage Examples

### Replace a Single Line
```json
{
  "path": "/project/src/config.js",
  "start_line": 15,
  "new_content": "const API_VERSION = 'v2';"
}
// Replaces line 15 with the new content
```

### Replace a Line Range
```json
{
  "path": "/project/src/header.js",
  "start_line": 10,
  "end_line": 15,
  "new_content": "// Header component rewritten\nimport React from 'react';\nimport { useState } from 'react';\n"
// Replaces lines 10 through 15 with the new content block
```

### Delete Lines
```json
{
  "path": "/project/src/temp.js",
  "start_line": 5,
  "end_line": 10,
  "action": "delete"
}
// Deletes lines 5 through 10 inclusive
```

### Insert Before a Line
```json
{
  "path": "/project/src/main.js",
  "start_line": 1,
  "action": "insert_before",
  "new_content": "// Auaci IDE Generated Code\n'use strict';\n"
// Inserts the new content before line 1 (at the beginning of file)
```

### Insert After a Line
```json
{
  "path": "/project/src/utils.js",
  "start_line": 20,
  "action": "insert_after",
  "new_content": "\nexport const helperFunctions = {\n  formatDate,\n  parseInput,\n  validateOutput\n};"
// Inserts the new content after line 20
```

### Clear File Contents (Delete All Lines)
```json
{
  "path": "/project/src/temp.txt",
  "start_line": 1,
  "end_line": 999999, // or use a large number
  "action": "delete"
}
// Deletes all lines from start to a large end number (effectively clears file)
```

## Best Practices
1. **Verify Line Numbers First**: Use `view` tool to examine file and confirm line numbers
2. **Use Ranges for Blocks**: Specify start_line and end_line for multi-line operations
3. **Consider Insert Points**: Remember that insert_before affects start_line, insert_after affects end_line
4. **Handle File Boundaries**: Be careful with line 1 (beginning) and last line (end of file)
5. **Empty Content for Deletion**: Omit new_content or set to empty string for delete actions
6. **Test Small Changes**: Start with single line changes before attempting range operations
7. **Use Descriptive Comments**: When inserting, consider adding comments to mark generated code

## Technical Implementation
The tool typically:
1. Reads the file into an array of lines
2. Performs splice operations based on line numbers and action
3. Joins the lines back together with appropriate line endings
4. Handles edge cases like inserting at beginning/end of file
5. Preserves original line ending style (LF/CRLF) when possible

## When to Use This Tool
- Adding headers or footers to files
- Inserting boilerplate code at specific locations
- Removing specific lines or blocks of code
- Replacing known problematic lines
- Making changes where line numbers are stable or known
- Working with generated code where line numbers are predictable
- Precise modifications in structured files (logs, configs, etc.)

## Limitations
- Requires accurate line numbers (can change if file is modified elsewhere)
- No content awareness - operates purely on line numbers
- Difficult to use in frequently changing files
- No automatic adjustment for inserted/deleted lines affecting subsequent line numbers
- Must calculate line numbers manually or through other tools

## Comparison with Similar Tools
- **vs str_replace**: Line-number-based vs content-based operations
- **vs apply_patch**: Precise line control vs context-based fuzzy matching
- **vs edit_function**: Arbitrary line ranges vs semantic function boundaries
- **vs batch_str_replace**: Single file line operations vs multiple string replacements

3. **Block Removal**: Removing deprecated code blocks
4. **Configuration Updates**: Changing specific lines in config files
5. **Test Modifications**: Adding or removing test cases at known locations
6. **Migration Scripts**: Updating version numbers or API endpoints on specific lines
7. **Template Filling**: Replacing placeholder lines in templates

## Tips for AI Agents
- Combine with `view` tool to get current line numbers before making changes
- Use relative line numbers when possible (e.g., "insert after line 5" rather than absolute)
- Consider making multiple small changes rather than one large range change
- Validate results with subsequent `view` or `str_replace` operations
- Be aware that line numbers shift after insertions/deletions
